## 1. 憲法判断：**改訂は不要**（これが今回の要点）

同日、憲法に抵触する提案が3回続いた（①コーチング再開 ②AI事業コンサル ③鑑定書）。今回は4回目に見えるが、**性質が違う**。

### 根拠：これは新商品ではなく、既にある椅子

| 憲法の条項 | 今回の扱い |
|---|---|
| Brand OS §3 収束の掟 | 受け皿を「**1. 既存の枠 = 顧問椅子**（顧問枠の空き次第）」と**明示的に許可**している |
| Brand OS §4 | 受け皿は「**顧問枠の空き次第**」＋過去在庫のみ、と§3と同一に規定 |
| Brand OS §2 ① | 「**相談役は同格の語なので可**」。顧問も可。禁止語は コーチ/コンサル/講師/メンター/指導/教える/支援者 |
| OS L4 今期やらないこと | 「新商品・**コーチング新規募集**」。顧問はコーチングではない |
| OS L4 今期の注力3 | 「**契約防衛: 既存4社の関係維持（月6時間の聖域）**」 |
| chiero.jp 事業02 | 「経営者の相談役 — **顧問として経営者の意思決定に伴走します**」＝**すでに公開済み** |

**決定的な一致（算術）**: L4 の「既存4社 / 月6時間の聖域」を割ると **4社 × 90分 = 360分 = 6時間**。
本人が今回指定した「**月1回で90分**」は、**いま動いている顧問契約とまったく同じ形**である。
→ 新しく作った商品ではない。**既存の枠に、値段と入り口を書いただけ**。

### L5起票（07-15）の分岐に照らす
起票時に自分で書いた分岐がそのまま効く：

> (a)「**顧問枠を開ける」だけなら改訂不要** —— Brand OSは「相談役」を同格の語として許可し、受け皿を「顧問枠の空き次第」と既に定めており、事業02として公開済み。**必要なのは問い合わせ導線のみ**。
> (b)「コーチングという商品を作って売る」なら4条項の改訂が必要。

本件は **(a)**。よって 8/1 の審議を待つ必要はない。**臨時発効も不要**（変える条文が無い）。

### §4「専用の面を作らない」との関係
§4の禁止対象は「**憧れ型需要の層**（1人でゆるく自由に生きたい）」への専用の面。
本件の相手は「**仕事を依頼したい人**」＝別の層。§4は直接かからない。

ただし§4の精神（**売り込み面を作った瞬間に信頼構造が壊れる**）は効かせる。
→ **説得しない。条件を開示するだけの面**として設計した。その具体：
- 価格を**全部出す**（非公開のハイチケット導線＝怪しさの構造そのものを避ける）
- **お客様の声を載せない**（§4／旧LPには6件あった）
- 限定N名・今だけ・先着 を**使わない**
- 「**向いていないこと**」を先に4つ書く（売り込みの反対）

## 4. 法務：**特商法表記が必須になった**

chiero.jp では意図的に置かなかった。その理由（原文）：
> 商品・価格・申込フォームを持たず通信販売に該当しないため、表記義務がない。**オンラインでの販売を始める時点で必須になる。**

**その引き金が引かれた。** 一次資料で確認した要点：

- **電話番号は省略不可**。消費者庁ガイド：「法人の場合には、名称、住所及び**電話番号を表示する必要があります**」「消費者が**容易に認識できる**文字の大きさ・場所に」
  → 本人は会社概要で「電話番号はなし」としたが、**この面では通らない**
- **法定返品権8日（15条の3）は適用されない**。同条は「**売買契約**＝商品・特定権利」が対象。顧問は**役務**。
  → ただし11条の「**契約の申込みの撤回・解除に関する事項**」の表示義務はある。「複数回又は一定期間の役務提供」＝**中途解約の方法を分かりやすく表示**する必要がある
- **特定継続的役務提供の7業種**（エステ/美容医療/語学/家庭教師/学習塾/パソコン教室/結婚相手紹介）に顧問は**非該当** → 法定クーリングオフなし
- **総額表示義務** → 税込で記載した（`50,000円（税込）` `1,100,000円（税込）`）。**税別なら要修正**
- 初回セッションの場で契約させない設計は、勧誘認定リスクの遮断も兼ねる

出典: [消費者庁 通信販売](https://www.no-trouble.caa.go.jp/what/mailorder/) / [通信販売広告について](https://www.no-trouble.caa.go.jp/what/mailorder/advertising.html) / [通信販売広告Q&A](https://www.no-trouble.caa.go.jp/qa/advertising.html)

## 7. 公開手順（3つの事実が揃ってから）

```bash
# 1. プレースホルダを実値に置換（3箇所）
#    CONTACT_TEL / CONTACT_EMAIL / CANCEL_POLICY
# 2. ゲートを通す（exit 0 が条件）
cd ~/Desktop/Claude/chiero_work && python3 tools/gate.py
# 3. リポジトリ作成 → push（Pagesはまだ有効化しない）
gh repo create kounkt/chiero-work --public --source=. --push
# 4. 【本人】Xserver DNS に work.chiero.jp CNAME kounkt.github.io を追加
# 5. 伝播を5台**全部**で確認（1台だけ見て誤判定した前科あり）
for ns in ns1 ns2 ns3 ns4 ns5; do dig +short CNAME work.chiero.jp @$ns.xserver.jp; done
# 6. DNSが通ってから CNAME ファイルを置いて Pages 有効化
#    ★鉄則: カスタムドメインをDNSより先に設定すると301ループで到達不能になる（同日に事故済み）
echo "work.chiero.jp" > CNAME && git add CNAME && git commit -m "custom domain" && git push
# 7. chiero.jp の事業02から work.chiero.jp へリンク（受け口は受けて初めて機能する）
```
